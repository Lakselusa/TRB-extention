1 Authenticate with Trimble Identity

Obtain an access token from the Trimble Identity OAuth2 flow. You'll need a registered application with the Trimble Connect scope. Paste the bearer token in the sidebar.

// Trimble Identity token endpoint POST https://id.trimble.com/oauth/token grant_type=authorization_code client_id=<your_app_id> redirect_uri=<your_redirect>

2 Select IFC Models to Combine

Once connected, the app fetches all files from your project. IFC files are highlighted and selectable. This mirrors the Trimble Connect Desktop workflow of loading multiple models into the 3D viewer before exporting — but entirely in the browser.

3 Export Combined .trb

Selected IFC files are downloaded via the TC Core API, then bundled into a single .trb container using the TrimBIM packaging format. The result downloads directly to your machine — no desktop app needed.

// How the Desktop app does it (inspiration): // 1. Load multiple IFC models in the 3D Viewer // 2. Click "Export current 3D view as TrimBIM" // 3. All loaded models get combined into one .trb // // This web app replicates that via the REST API: // 1. GET /files → list project files // 2. GET /files/{id}/content → download each IFC // 3. Bundle into .trb container client-side // 4. Trigger browser download

4 Alternative: Use the Workspace API

For projects that want deeper integration, embed the TC 3D Viewer as an iframe and use the Workspace API's getTrimbimModels() and addTrimbimModel() methods to programmatically access the converted TrimBIM data that TC's servers generate from IFC files.

// Embed the 3D viewer and use the Workspace API const API = await WorkspaceAPI.connect(iframe); API.embed.setTokens({ accessToken }); API.embed.init3DViewer({ projectId }); // Load models → TC converts IFC → TrimBIM server-side await API.viewer.toggleModel(modelId, true); // Retrieve the converted trimbim data const trbs = await API.viewer.getTrimbimModels();

Trimble Connect Core API — File Endpoints

These are the key REST endpoints used by this application to list and download files from a Trimble Connect project.

// List all files in project GET {region}/tc/api/2.0/projects/{projectId}/files Authorization: Bearer <token> // Get file metadata GET {region}/tc/api/2.0/files/{fileId} // Download file content GET {region}/tc/api/2.0/files/{fileId}/content // Get TrimBIM version (if already converted) GET {region}/tc/api/2.0/files/{fileId}/trimbim

Workspace API — Viewer Methods

When embedding the 3D Viewer, these methods interact with loaded TrimBIM models directly.

// Get all loaded TrimBIM models viewer.getTrimbimModels() → Promise<TrimbimModel[]> // Add a TrimBIM model (max 10MB per blob) viewer.addTrimbimModel(model) → Promise<boolean[]> // Remove a TrimBIM model viewer.removeTrimbimModel(id) → Promise<void> // Toggle model loading (triggers IFC→TRB conversion) viewer.toggleModel(modelId, load?) → Promise<void> // Get list of all available models viewer.getModels() → Promise<Model[]>

TrimBIM (.trb) Format Notes

TrimBIM is Trimble's proprietary binary format optimized for fast random access of BIM geometry and attributes. The conversion from IFC to TrimBIM is performed server-side by Trimble Connect when a model is first loaded in the 3D Viewer. The .trb file stores the same IFC data in a more efficient layout.
